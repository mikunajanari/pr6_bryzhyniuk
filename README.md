# Алгоритмічні та евристичні методи обчислення площі геометричних фігур
## Опис
1. Shapely

Бібліотека Shapely використовує оптимізовані геометричні алгоритми (GEOS C++). Її результат використовується як еталонне значення площі.

2. Метод Гауса (Shoelace Formula)

Аналітичний метод обчислення площі багатокутника за координатами його вершин. Має високу швидкість та точність.

3. Метод Монте-Карло

Чисельний ймовірнісний метод. Усередині обмежувального прямокутника генеруються випадкові точки. Частка точок, що потрапили в полігон, використовується для оцінки площі.

Метод Монте-Карло є випадковим, тому результат кожного запуску залежить від випадково згенерованих точок. Збільшення кількості точок зазвичай підвищує точність, але не гарантує, що кожен конкретний запуск буде кращим за попередній. Наприклад, для 100 точок похибка може виявитися трохи більшою, ніж для 10, якщо випадковий розподіл точок цього разу був менш вдалим біля меж полігона. Похибка зменшується в середньому, а не строго монотонно після кожного збільшення кількості точок. Чим більше точок використовується, тим меншими стають такі коливання, але повністю вони не зникають.

## Висновки
Метод Гауса показав дуже високу швидкість роботи та точний результат, практично ідентичний еталонному значенню Shapely. Це робить його найкращим вибором для задач обчислення площі полігонів за відомими координатами вершин.

Метод Монте-Карло працює значно повільніше, оскільки потребує генерації великої кількості випадкових точок та перевірки їх входження в полігон.

За результатами експерименту видно, що зі збільшенням кількості точок похибка зменшується. Для прийнятної точності достатньо приблизно 10000 точок, оскільки похибка вже становить менше 1%. Подальше збільшення кількості точок підвищує стабільність, але значно збільшує час обчислень.
